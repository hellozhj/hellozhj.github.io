ES6 新特性

声明变量的关键字区别：var 与 let 的区别

ES5 声明变量关键字：var

  1. 可以先使用后声明 var 有预解析

    a = 10;
console.log(a);
var a;

  2. 可以重复声明

    var a = 10;
var b = 20;
console.log(a);

  3. 在for循环中使用对全局变量造成的污染问题

    for (var i = 0; i < 5; i++) {
    // console.log(i);
}
console.log(i);		//5

  4. 作用域：全局和函数 var 没有块级作用域

    {
    var a = 10;     // 全局变量
}
console.log(a)		//10

**ES6 声明变量的关键字：let **

  1. 不可以先使用后声明 Cannot access ‘a’ before initialization

    console.log(a);
let a = 10;

  2. 不可以重复声明 let 没有预解析

    let a = 10;
let a = 20;

  3. 不会在for循环中使用对全局变量造成污染问题

    for (let i = 0; i < 5; i++) {
    // console.log(i);
}
console.log(i);		//i is not defined

  4. {}就是块级作用域

    {
    let a = 10;
}
console.log(a);		//a is not defined

页面有5个按钮，点击每一个按钮打印对应的索引

ES5 原生

var btns = document.getElementsByTagName("button");
console.log(btns);
for (var i = 0; i < btns.length; i++) {

    // 存索引(标签属性)
    btns[i].setAttribute("myIndex", i);

    // 存索引(对象属性)
    btns[i].myIndex = i;

    btns[i].onclick = function () {
        // 事件的绑定是异步，点击的时候for循环早就已经执行完毕，此时的i永远变成了5
        console.log(i);

        console.log(this.getAttribute("myIndex"));

        console.log(this.myIndex);
    }
}

ES6

let btns = document.getElementsByTagName("button");
for (let i = 0; i < btns.length; i++) {
    // {i=0 btns[0]     0}      形成原因就是块级作用域 i=0 的 i 和 i=1 的 i 不是同一个变量
    // {i=1 btns[1]     1}
    // {i=2 btns[2]     2}
    // {i=3 btns[4]     0}
    // {i=4 btns[4]     4}
    btns[i].onclick = function () {
        console.log(i);
    }
}

Promise.resolve 方法的参数如果是一个原始值，或者是一个不具有 then 方法的对象，则 Promise.resolve 方法返回一个新的 Promise 对象，状态为resolved，Promise.resolve 方法的参数，会同时传给回调函数。

then 方法接受的参数是函数，而如果传递的并非是一个函数，它实际上会将其解释为 then(null)，这就会导致前一个 Promise 的结果会穿透下面。

快速排序

快速排序： 相对于冒泡减少了没必要的次数 也能达到排序的效果

实现思路:

  • 找出数组的中间值，取出来，改变了原数组
  • 将已经改变之后的数组进行循环遍历和这个取出来的中间值进行比较，小于中间值的 都统一打包放在一个数组中(l),反之放在另外一个数组中(r) [l mid r]最后将这些数组进行合并得到一个新的数组

… 展开语法

function quick(newArr) {
    // 出口
    if (newArr.length <= 1) {
        return newArr;
    }
    // 拿到中间数
    var mid = newArr.splice(Math.floor(newArr.length / 2), 1)[0];
    // console.log(mid);
    // console.log(arr);
    // 对这个已经改变的数组进行循环遍历
    let l = [], r = [];
    newArr.forEach(item => {
        // 1:每个数据项和中间值进行比较  小的放在l数组中    大的放在r数组中
        if (item < mid) {
            l.push(item)
        } else {
            r.push(item)
        }
    })
    // console.log(1, mid, r);
    // debugger

    // ... 展开语法 
    // return [...quick(1), mid, ...quick(r)]

    return quick(l).concat(mid, quick(r))
}

JSON数据格式

定义：JSON 格式（JavaScript Object Notation 的缩写）是一种用于前后端数据交换的文本格式。

JSON 对值的类型和格式有严格的规定：

  1. 复合类型的值只能是数组或对象，不能是函数、正则表达式对象、日期对象。
  2. 原始类型的值只有四种：字符串、数值（必须以十进制表示）、布尔值和null（不能使用NaN, Infinity, -Infinity和undefined）。
  3. 字符串必须使用双引号表示，不能使用单引号。
  4. 对象的键名必须放在双引号里面。
  5. 数组或对象最后一个成员的后面，不能加逗号。

Event Loop即事件循环，是指浏览器或Node的一种解决javaScript单线程运行时不会阻塞的一种机制，也就是我们经常使用异步的原理。

宏任务：
1：script(所有的同步代码)1,5,7是属于同步代码    在script环境中是一瞬间完成的
2：如果页面只剩下异步代码  会先找微任务列队  输出了6     微任务没了
3：定时器(宏任务里面的异步代码) 2和3相对于定时器环境中属于同步代码  一瞬间完成
4：定时器中的.then 是 宏任务列队中的微任务，先会执行，执行之后会继续在宏任务列队检查是否还有未执行的代码，如果有，按照任务列队的执行顺序继续寻找微任务，最后执行宏任务

宏任务和微任务的执行顺序是先宏后微。先执行同步代码，遇到异步宏任务则将异步宏任务放入宏任务队列中，遇到异步微任务则将异步微任务放入微任务队列中，当所有同步代码执行完毕后，再将异步微任务从队列中调入主线程执行，微任务执行完毕后再将异步宏任务从队列中调入主线程执行，一直循环直至所有任务执行完毕。
